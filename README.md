# PoopSeek

> 🚀 AI-агент в CLI с доступом к инструментам файловой системы и shell

PoopSeek — это интерактивный командный агент, построенный на базе **DeepSeek Chat API** и **Bun**. Агент умеет выполнять операции с файлами, запускать bash/powershell команды и самостоятельно принимать решения о необходимых действиях.

## ✨ Особенности

- 🤖 **Agent Loop** — автономное планирование и выполнение действий
- 🛠️ **8 встроенных инструментов**: bash, powershell, file.read/write/edit/list/remove, tools.list
- 🎨 **Цветной CLI** с поддержкой тем и автодополнением команд
- 📝 **Управление контекстом** — до 40 сообщений, промпты из внешних файлов
- 🔄 **Переменные в промптах** — динамическая подстановка значений
- 🧠 **DeepSeek API** — современная LLM с поддержкой функций (function calling)
- ⚡ **Высокая производительность** — Bun runtime, TypeScript

## 🚀 Быстрый старт

### Требования

- [Bun](https://bun.sh) (v1.0 или новее)
- User Token в DeepSeek (получить на [chat.deepseek.com](https://chat.deepseek.com)) через `localStorage.getItem('userToken')`

### Установка

```bash
# Клонирование репозитория
git clone <your-repo-url>
cd poopseek

# Установка зависимостей
bun install

# Настройка окружения
cp .env.example .env  # если есть пример
# Или создайте .env вручную:
echo "DEEPSEEK_TOKEN=ваш_ключ_api" > .env
```

### Запуск

```bash
bun start
```

Или в режиме разработки:

```bash
bun run --watch src/index.ts
```

### Сборка exe

```bash
bun run build:exe --version 1.2.3
```

Если `--version` не передан, `exe` использует `POOPSEEK_VERSION` или `APP_VERSION`, а затем `dev`.

## 🎮 Использование

### Интерактивный режим

После запуска вы увидите приглашение `>`. Вводите запросы на естественном языке — агент сам решит, какие инструменты вызвать.

**Пример диалога:**

```
> создай файл test.txt с текстом "Hello World"
🔨 Выполняется: file-write
[tool] file-write ok=true
Файл test.txt успешно создан.

> покажи содержимое файла
🔨 Выполняется: file-read
[tool] file-read ok=true
Содержимое: Hello World

> сколько файлов в текущей папке?
🔨 Выполняется: bash
[tool] bash ok=true
Всего файлов: 12
```

### Встроенные команды CLI

Все команды начинаются с `/`:

| Команда | Описание |
|---------|----------|
| `/help` | Показать список всех команд |
| `/clear` | Очистить историю контекста (начать новый диалог) |
| `/session` | Показать ID текущей сессии |
| `/stats` | Показать статистику контекста (количество сообщений) |
| `/theme [dark\|light]` | Сменить цветовую тему |
| `/exit` или `/quit` | Выйти из приложения |

**Подсказки**: При вводе нажимайте `Tab` для автодополнения команд. Во время ввода отображаются подсказки доступных команд.

### Доступные агенту инструменты

Агент может использовать следующие инструменты (описаны в `docs/tools/tool.base.md`):

| Инструмент | Назначение |
|------------|------------|
| `bash` | Выполнение bash-команд (Linux/macOS/WSL) |
| `powershell` | Выполнение PowerShell команд (Windows) |
| `file.read` | Чтение содержимого файла |
| `file.write` | Создание или перезапись файла |
| `file.edit` | Частичное редактирование файла (замена строк) |
| `file.list` | Получение списка файлов в директории |
| `file.remove` | Удаление файла или директории |
| `tools.list` | Просмотр доступных инструментов |

Агент автоматически выбирает нужные инструменты на основе вашего запроса.

### Ключевые компоненты

#### 1. Agent Loop (`src/agent/loop.ts`)

Основной цикл принятия решений:
- Получает запрос пользователя
- Добавляет его в контекст
- Запрашивает у DeepSeek следующий шаг (текст или вызов инструмента)
- Если инструмент — выполняет через `ToolExecutor` и продолжает цикл
- Максимум 12 шагов на один пользовательский ввод

#### 2. Context Manager (`src/agent/context-manager.ts`)

Управляет историей диалога:
- Хранит до 40 последних сообщений
- Внедряет системный промпт и описание инструментов
- Поддерживает переменные (например, `{{workspaceRoot}}`)
- Может очищать историю по команде `/clear`

#### 3. Tool Executor (`src/agent/tool-executor.ts`)

Безопасное выполнение инструментов:
- Валидация аргументов по JSON Schema
- Изоляция операций в рабочей директории
- Единый интерфейс для всех инструментов (`{ ok, output, error }`)

#### 4. DeepSeek Client (`src/deepseek-client/`)

Абстракция над DeepSeek API:
- Управление сессиями (поддержка conversation history)
- Function calling для вызова инструментов
- Потоковая передача ответов (streaming)

## ⚙️ Конфигурация

```env
DEEPSEEK_TOKEN=sk-...   # Обязательно: API ключ DeepSeek
```

### Кастомизация промптов

Вы можете редактировать эти файлы для изменения поведения агента.

### Переменные в промптах

Поддерживаются переменные вида `{{variableName}}`, которые обрабатываются через `src/variables/`. Например:
- `{{workspaceRoot}}` — текущая рабочая директория

## 🧪 Разработка

### Сборка TypeScript

Проект использует Bun как runtime, отдельная компиляция не требуется — Bun выполняет TypeScript нативно.

### Добавление нового инструмента

1. Создайте файл в `src/tools/`, например `my-tool.ts`
2. Реализуйте интерфейс:
   ```typescript
   export const myTool: Tool = {
       name: "my-tool",
       description: "Описание для LLM",
       parameters: {
           type: "object",
           properties: {
               param1: { type: "string" }
           }
       },
       execute: async (args) => {
           // логика выполнения
           return { ok: true, output: "результат" };
       }
   };
   ```
3. Зарегистрируйте инструмент в `src/tools/index.ts`
4. Добавьте описание в `docs/tools/tool.base.md` для LLM

### Отладка

Включите логирование, добавив в `run-cli.ts`:
```typescript
process.env.DEBUG = "true";
```
