# Управление контекстом и сжатие истории

## Обзор

PoopSeek поддерживает длительные диалоги через управление контекстом и автоматическое/ручное сжатие истории. Система предотвращает переполнение контекстного окна DeepSeek.

## Контекстный менеджер (`context-manager.ts`)

### Основные функции

```typescript
class ContextManager {
    // Добавление сообщения в историю
    addMessage(message: Message): void
    
    // Получение текущей истории для отправки в API
    getMessages(): Message[]
    
    // Сжатие истории через LLM
    async compact(keepLast?: number): Promise<void>
    
    // Очистка истории
    clear(): void
    
    // Сохранение/загрузка сессии
    save(): Promise<void>
    load(): Promise<void>
}
```

### Ограничения

- **Максимум сообщений**: 40 по умолчанию (настраивается в `runtime-config.ts`)
- **При превышении**: автоматически сжимает старые сообщения, оставляя последние 20
- **Лимит токенов**: отслеживается приблизительно (1 токен ≈ 0.75 слова на русском)

### Типы сообщений

```typescript
// Системный промпт (всегда первым)
{ role: "system", content: "You are..." }

// Пользовательский ввод
{ role: "user", content: "Hello!" }

// Ответ ассистента
{ role: "assistant", content: "Hi there!" }

// Результат выполнения инструмента
{ role: "tool", content: "ls: file.txt", tool_call_id: "call_123" }
```

## Механизм compact

### Как работает

1. Пользователь вызывает `/compact` или система автоматически решает сжать историю
2. Контекстный менеджер отправляет запрос к DeepSeek: *"Resume this conversation in 10 sentences, keep key facts and user requests"*
3. Модель возвращает краткое резюме
4. История заменяется на: системный промпт + резюме + последние N сообщений

### Автоматический compact

Срабатывает при:
- Превышении лимита в 40 сообщений
- Приближении к лимиту токенов (80% от 128K)
- Инструментальные вызовы занимают много места (например, большие выводы команд)

### Ручной compact

```bash
# Сжать всю историю (рекомендуется, если диалог длинный)
/compact

# Сжать только последние 20 сообщений, оставив более старые
/compact 20
```

### Пример сжатия

**До compact (15 сообщений):**
```
[system] You are a Python expert
[user] Как создать список?
[assistant] Используйте [1, 2, 3]
[user] А как добавить элемент?
[assistant] Используйте .append()
... ещё 10 сообщений про Python
```

**После compact (3 сообщения):**
```
[system] You are a Python expert
[system compact] Пользователь учит Python: списки, .append(), словари, циклы for, функции. Последние вопросы про понимание списков.
[user] А как работает list comprehension?
```

## Сохранение сессий

### Автосохранение
- После каждого сообщения история сохраняется в `~/.poopseek/sessions/last.json`
- При запуске восстанавливается последняя сессия
- Можно сохранить несколько сессий через `/session save <name>`

### Управление сессиями

```bash
/session save myproject   # сохранить текущую сессию как "myproject"
/session load myproject   # загрузить сохранённую сессию
/session list             # показать все сохранённые сессии
/session delete myproject # удалить сессию
```

## Влияние инструментов на контекст

Инструментальные вызовы и их результаты занимают много места:

```json
// Вызов инструмента (маленький)
{ "role": "assistant", "tool_calls": [...] }

// Результат (может быть огромным)
{ "role": "tool", "content": "1000 строк лога" }
```

Рекомендации:
- Используйте `/compact` после серии из 5-10 инструментальных вызовов
- Ограничивайте вывод команд через `head -n 20`
- Для больших файлов используйте `file.read` с аргументом `limit`

## Настройка лимитов

В `src/agent/context-manager.ts`:

```typescript
const DEFAULT_MAX_MESSAGES = 40;
const MAX_COMPACT_KEEP = 20;        // после сжатия оставлять 20 последних
const TOKEN_BUFFER = 10000;         // буфер для инструментов
```

Для изменения отредактируйте файл и пересоберите проект.

## Отладка контекста

```bash
/debug context   # показать текущую историю (первые 100 символов каждого сообщения)
/debug tokens    # показать приблизительное количество токенов
```

## Известные проблемы

- **Точность подсчёта токенов**: используется эвристика, может отличаться от реального API
- **Потеря контекста при сжатии**: модель может упустить важные детали, используйте `/compact` осторожно
- **Долгое сжатие**: при большой истории (30+ сообщений) compact может занимать 10-30 секунд
