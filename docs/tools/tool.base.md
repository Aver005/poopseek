Вызов инструмента ВСЕГДА происходит в JSON формате.
Пример вызова инструмента для просмотра всех инструментов:

```json
{
    "tool": "tools.list",
    "args": {},
    "onError": "ignore",
    "onSuccess": "continue",
}
```

`onError` - Что делать при неудачном вызове
- `continue` - продолжить выполнение
- `stop` - остановить выполнение
- `try-again` - попробовать снова
- `ignore` - игнорировать ошибку
- `ask-user` - спросить пользователя что делать

`onSuccess` - Что делать при успешном вызове
- `continue` - продолжить выполнение
- `stop` - остановить выполнение
- `try-again` - попробовать снова
- `ask-user` - спросить пользователя что делать

## Инструмент: git

Выполняет git-команду в корне рабочего пространства.

**Аргументы:**
- `command` (string, обязательный) — git-подкоманда и флаги (без префикса `git`)

**Примеры:**
```json
{
    "tool": "git",
    "args": { "command": "status" },
    "onError": "continue",
    "onSuccess": "continue"
}
```
```json
{
    "tool": "git",
    "args": { "command": "log --oneline -10" },
    "onError": "stop",
    "onSuccess": "continue"
}
```
```json
{
    "tool": "git",
    "args": { "command": "diff HEAD" },
    "onError": "stop",
    "onSuccess": "continue"
}
```

## Инструмент: git.edit

Применяет unified diff patch к файлам в рабочем пространстве.
Парсит патч и применяет хунки программно — без вызова `git apply`.
Перед применением проверяет совпадение контекстных и удаляемых строк.

**Аргументы:**
- `patch` (string, обязательный) — unified diff в стандартном формате

**Формат патча:**
- Заголовок файла: `--- a/path/to/file` / `+++ b/path/to/file`
- Хунк: `@@ -oldStart,oldCount +newStart,newCount @@`
- Строки: ` ` контекст, `-` удалить, `+` добавить
- Пути относительно корня рабочего пространства

**Пример:**
```json
{
    "tool": "git.edit",
    "args": {
        "patch": "--- a/src/foo.ts\n+++ b/src/foo.ts\n@@ -10,4 +10,4 @@\n function greet() {\n-    return 'hello';\n+    return 'hi';\n }\n"
    },
    "onError": "stop",
    "onSuccess": "continue"
}
```

**Важно:**
- Контекстные строки (` `) и удаляемые (`-`) должны точно совпадать с содержимым файла
- Пути в патче должны указывать на файлы внутри рабочего пространства
- Для просмотра текущего diff перед редактированием используй `git diff`
