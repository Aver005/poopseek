# Команды пользователя

## Обзор

PoopSeek поддерживает slash-команды для управления сессией, моделью, провайдерами, навыками и настройками. Команды вводятся в начале строки и обрабатываются до отправки в LLM.

## Список команд

### `/help`
Показывает справку по всем доступным командам.

### `/tools`
Выводит список доступных инструментов с описаниями.

### `/model [name]`
Просмотр или смена модели.

```
/model          # показать текущую модель
/model expert   # переключиться на expert
```

### `/switch [default|expert]`
Быстрое переключение модели между default и expert. Без аргументов — переключает на противоположную.

### `/provider [id]`
Просмотр или смена LLM-провайдера. Без аргументов показывает текущего и список настроенных. С аргументом — переключает на указанного.

Доступные провайдеры: `deepseek-web`, `openai`, `openrouter`, `hugging-face`, `claude`, `gemini`, `ollama`, `lm-studio`.

### `/auth`
Интерактивная настройка аккаунта: имя пользователя, выбор провайдеров, ввод API-ключей.

### `/logout`
Сбросить токен и выйти из приложения.

### `/relogin`
Ввести новый токен без перезапуска.

### `/compact [N]`
Сжать историю диалога через LLM.

```
/compact      # сжать всю историю
/compact 20   # сжать последние 20 сообщений
```

### `/cmpt [N]`
Алиас для `/compact`.

### `/clear`
Очистить историю диалога.

### `/history`
Очистить историю диалога (аналогично `/clear`).

### `/theme [name]`
Управление цветовой темой.

```
/theme        # показать текущую тему
/theme dark   # тёмная тема (по умолчанию)
/theme light  # светлая тема
```

### `/btw <question>`
Sidechat — фоновый запрос без прерывания основного диалога.

### `/exit` или `/quit`
Завершить работу программы.

### `/session`
Показать информацию о текущей сессии.

### `/sessions`
Открыть список локально сохранённых сессий с возможностью загрузки.

### `/load <session_id>`
Загрузить сессию по ID (локальную или из DeepSeek).

### `/stats`
Показать статистику контекста (количество сообщений, токенов).

### `/think`
Переключить режим рассуждений (thinking_enabled).

### `/web`
Переключить веб-поиск (search_enabled).

### `/maestro`
Активировать все навыки (алиас `/skills all`).

### `/noob`
Деактивировать все навыки (алиас `/skills reset`).

### `/skills [list [page] | all | reset | use <name> | off <name> | clear]`
Управление навыками. Без аргументов открывает интерактивный список.

- `/skills list` — постраничный список (Enter для интерактивного режима)
- `/skills list 2` — страница 2 списка
- `/skills all` — активировать все
- `/skills reset` — деактивировать все
- `/skills use <name>` — активировать конкретный навык
- `/skills off <name>` — деактивировать конкретный навык
- `/skills clear` — деактивировать все

### `/skills-folder [list | add <путь> | remove | reset]`
Управление дополнительными папками с навыками.

### `/mcp [list | tools | resources | prompts | connect | disconnect | enable | disable | reload | info | read | prompt]`
Управление MCP-серверами Model Context Protocol.

- `/mcp` — статус всех серверов
- `/mcp list` — список серверов с деталями
- `/mcp tools [server]` — список MCP-инструментов
- `/mcp resources [server]` — список MCP-ресурсов
- `/mcp prompts [server]` — список MCP-промптов
- `/mcp connect <server>` — подключить/переподключить сервер
- `/mcp disconnect <server>` — отключить сервер
- `/mcp enable <server>` — включить отключённый сервер
- `/mcp disable <server>` — отключить и заморозить сервер
- `/mcp reload` — перезагрузить конфиги и переподключиться
- `/mcp info <server>` — подробная информация о сервере
- `/mcp read <server> <uri>` — прочитать MCP-ресурс
- `/mcp prompt <server> <name>` — получить MCP-промпт

### `/refactor <easy|medium|hard|ultra> [фокус]`
Запуск рефакторинга кода с выбранным уровнем интенсивности.

### `/review [staged | branch [ref] | commit <ref> | file <path>]`
Код-ревью изменений.

## Синтаксис и особенности

### Многострочные команды
Команды с длинными аргументами (например, `/btw`) могут быть продолжены на следующей строке через Shift+Enter.

### Аргументы с пробелами
Если аргумент содержит пробелы, можно использовать кавычки:

```
/model "my-custom-model"
```

### Автокомплит команд
При вводе `/` терминал предлагает доступные команды (через Tab).

## Добавление новой команды

См. `development-guide.md` → раздел "Добавление slash-команды".

## Поведение при ошибках

- **Неизвестная команда**: выводит "Неизвестная команда. Введите /help для списка команд."
- **Неверные аргументы**: выводит usage для конкретной команды
- **Команда во время генерации**: ставится в очередь, выполнится после завершения текущего ответа
