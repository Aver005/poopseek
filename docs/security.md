# Безопасность и ограничения

## Обзор

PoopSeek выполняет инструменты (bash, file операции) от имени пользователя. Неправильное использование может привести к потере данных или уязвимостям. Следуйте этому руководству для безопасной работы.

## Основные риски

### 1. Выполнение команд
Модель может вызвать `bash` или `powershell` с произвольными командами.

**Потенциальные проблемы:**
- Удаление файлов (`rm -rf`, `del /F`)
- Изменение системных настроек
- Скачивание и запуск вредоносного кода
- Отправка данных по сети

**Защита:**
- Агент работает **только в текущей рабочей директории** и её подпапках
- Команды не могут выходить за пределы workspace через `..`
- Нет доступа к системным директориям (Windows: `C:\Windows`, Unix: `/etc`, `/usr`)

### 2. Чтение и запись файлов
Модель может читать/писать/удалять файлы в workspace.

**Потенциальные проблемы:**
- Чтение секретов (`.env`, ключи API, пароли)
- Перезапись важных файлов
- Утечка кода через загрузку на внешние серверы

**Защита:**
- `.gitignore` автоматически уважается (не читает игнорируемые файлы без флага `--force`)
- Логирование всех файловых операций в `~/.poopseek/audit.log`
- Подтверждение перед перезаписью существующих файлов (зависит от режима)

### 3. Инъекции в инструменты
Модель может передать вредоносные аргументы.

**Пример:** `file.read` с путём `../../../etc/passwd`

**Защита:**
- Валидация всех путей через `path.resolve()` и проверка на `workspaceRoot`
- Экранирование аргументов в `bash` (используется `JSON.stringify`)
- Ограничение длины вывода команд (максимум 100KB для безопасности)

## Ограничения безопасности

### По умолчанию
- **Рабочая директория**: только `C:\Work\Me\poopseek` (и её подпапки)
- **Запрещённые пути**: `..`, `~`, `C:\Windows`, `/etc`, `$HOME`
- **Максимальный размер вывода**: 1MB для `bash`, 10MB для `file.read`
- **Таймауты**: команды bash ограничены 30 секундами
- **Сетевой доступ**: модель не может инициировать сетевые запросы (кроме API DeepSeek)

### Режимы безопасности

Реализованы три уровня (выбор через флаг `--security` при запуске):

```bash
poopseek --security strict   # всё подтверждать (по умолч.)
poopseek --security normal   # подтверждать только опасные операции
poopseek --security permissive # доверять модели (не рекомендуется)
```

| Уровень | Удаление | Запись вне workspace | Сетевые вызовы | Чтение .env |
|---------|----------|---------------------|----------------|-------------|
| strict  | Запрещено | Запрещено           | Запрещено      | Запросить   |
| normal  | Запросить | Запрещено           | Запрещено      | Разрешено   |
| permissive | Разрешено | Разрешено        | Разрешено      | Разрешено   |

## Аудит и логи

Все действия логируются в `~/.poopseek/audit.log`:

```json
{
    "timestamp": "2024-01-15T10:30:00Z",
    "tool": "bash",
    "args": { "command": "rm important.txt" },
    "result": "success",
    "user_approved": true,
    "session_id": "abc123"
}
```

Просмотр логов:

```bash
cat ~/.poopseek/audit.log | jq '.'
```

## Рекомендации для пользователей

### DO (делайте)
- Работайте в изолированном проекте
- Используйте `--security strict` для важных данных
- Регулярно проверяйте `audit.log`
- Делайте бэкапы перед сложными операциями
- Используйте `/compact` для очистки истории

### DON'T (не делайте)
- Не запускайте в системных директориях (`/`, `C:\`)
- Не доверяйте модели критичные данные (пароли, ключи)
- Не игнорируйте запросы подтверждения
- Не оставляйте сессию без присмотра

## Безопасная работа с секретами

### Автоматическое исключение
Файлы с секретами автоматически не читаются без явного разрешения:
- `.env`
- `*.key`, `*.pem`
- `secrets.json`, `config/secrets/*`
- `~/.ssh/*`

### Как обойти
```bash
# Требуется подтверждение вручную
> Покажи содержимое .env
Are you sure you want to read .env? (y/n): y
```

## Уязвимости и компрометации

### Что делать при подозрении
1. **Немедленно завершите сессию** (`Ctrl+C` дважды или `/exit`)
2. **Проверьте аудит-лог** (`~/.poopseek/audit.log`)
3. **Отзовите токен DeepSeek** (если он был скопирован)
4. **Удалите подозрительные файлы** (если созданы)
5. **Сообщите об инциденте** (если нашли уязвимость)

### Известные ограничения защиты
- **Нет песочницы**: команды выполняются с правами текущего пользователя
- **Нет изоляции процессов**: `bash` может запустить долгоиграющие процессы
- **Нет контроля сети**: модель может вызвать `curl`, `wget` (но в `strict` режиме запрещено)
- **Windows особенности**: PowerShell может обходить ограничения путей через `Get-Content -Path "C:\\Windows\\System32\\config\\SAM"` (заблокировано через валидацию пути)
