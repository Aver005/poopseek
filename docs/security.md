# Безопасность

## Текущее состояние

На данный момент PoopSeek находится в стадии активной разработки. Механизмы безопасности, описанные ниже, являются **планируемыми** и ещё не реализованы в коде.

## Планируемые механизмы

### Режимы безопасности
Планируется флаг `--security` с режимами:
- `strict` — максимальные ограничения, подтверждение для всех операций записи
- `normal` — стандартные ограничения (по умолчанию)
- `permissive` — минимальные ограничения (для доверенных сред)

### Защита файлов
- Проверка `.gitignore` перед чтением файлов
- Автоматическое исключение файлов секретов: `.env`, `*.key`, `secrets.json` и др.
- Подтверждение перед перезаписью существующих файлов

### Ограничения выполнения
- Максимальный размер вывода bash/powershell: 1MB
- Максимальный размер file.read: 10MB
- Таймаут команд bash: 30 секунд

### Аудит
- Логирование действий в `~/.poopseek/audit.log`
- Запись: временная метка, инструмент, аргументы, результат (ok/error)

## Что уже работает

- `resolvePath()` в инструментах валидирует пути относительно workspace root
- `user.confirm` — инструмент для явного запроса подтверждения у пользователя
- Команда `/quit` (и `/exit`) корректно завершает работу

## Рекомендации

- Не передавайте секреты в сообщениях модели
- Используйте `.env` для хранения токенов (файл в `.gitignore`)
- Проверяйте вывод команд перед применением изменений
