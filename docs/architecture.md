# Архитектура PoopSeek

## Обзор

PoopSeek — это CLI-агент на базе DeepSeek с поддержкой инструментов (tools). Написан на TypeScript, работает на Bun.

## Основные компоненты

### 1. CLI-слой (`src/cli/`)
- `run-cli.ts` — точка входа, оркестрация всех компонентов
- `terminal-input.ts` — обработка ввода с автокомплитом файлов
- `markdown.ts` — рендер markdown в терминале
- `colors.ts` — управление цветами и темами
- `runtime-config.ts` — хранение токена и версии

### 2. Агент (`src/agent/`)
- `loop.ts` — основной цикл: запрос к LLM → парсинг вызовов инструментов → выполнение → продолжение
- `context-manager.ts` — управление историей сообщений (максимум 40 сообщений, поддержка compact)
- `tool-executor.ts` — диспетчер вызовов инструментов
- `tool-call-parser.ts` — извлечение вызовов инструментов из ответа модели

### 3. Инструменты (`src/tools/`)
Определены в `defs/`. Каждый инструмент имеет:
- Имя и описание для модели
- Схему аргументов (JSON Schema)
- Обработчик выполнения

### 4. DeepSeek Client (`src/deepseek-client/`)
- Обёртка над API DeepSeek
- Поддержка сессий
- Потоковая передача ответов

### 5. Команды (`src/commands/`)
Обработчики slash-команд: `/help`, `/tools`, `/model`, `/theme`, `/compact` и т.д.

### 6. Переменные (`src/variables/`)
Подстановка переменных в промпты (например, `{{workspace}}`).

### 7. Bridge (`src/bridge/`)
Утилиты для работы с потоками DeepSeek.

## Поток данных

```
Пользовательский ввод
    ↓
terminal-input (обработка @файлов, многострочность, очередь)
    ↓
handleCommand (если /команда)
    ↓
agent-loop.runTurn()
    ↓
contextManager → DeepSeek API (stream)
    ↓
parseToolCalls()
    ↓
toolExecutor.execute()
    ↓
результат → обратно в loop (если нужно продолжить)
    ↓
renderMarkdown() → вывод
```

## Ключевые особенности

### Очередь ввода
Во время генерации ответа новый ввод не теряется, а добавляется в очередь.

### Файловые упоминания
Синтаксис `@path/to/file` автоматически читает файлы и добавляет их в контекст.

### Sidechat (`/btw <вопрос>`)
Фоновый запрос без прерывания основного диалога.

### Compact
Сжатие истории (`/compact`) — модель резюмирует диалог, освобождая место в контексте.

## Зависимости

- **Bun** — runtime и сборщик
- **terminal-kit** — продвинутый терминальный ввод
- **TypeScript** — типизация

## Сборка

```bash
bun run build      # собрать в build/
bun run build:exe  # скомпилировать в .exe
bun run start      # запуск в dev-режиме
```

## Конфигурация

- `.env` — переменные окружения (`DEEPSEEK_TOKEN`)
- `.poopseek.json` (автосоздаётся) — сохранённый токен и версия
- `assets/prompts/` — системные промпты
